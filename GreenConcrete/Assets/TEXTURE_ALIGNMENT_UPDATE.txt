HOW IT WORKS
------------

1. When a terrain tile is created, the system calculates its
   position in world space

2. Based on that position and the texture's tileSize, it
   computes the proper texture offset needed

3. A per-tile copy of each TerrainLayer is created with the
   adjusted offset values

4. The result: textures appear to continue naturally from
   one tile to the next

TROUBLESHOOTING
---------------

Q: "Textures still don't match at borders"
A: Ensure your textures are properly tileable. Check that all
   terrain tiles have the same size. Verify texture tileSize
   is consistent.

Q: "Some tiles align, others don't"
A: Each biome's textures align independently. If you walk from
   Forest to Plains, there will be a transition - but within
   each biome, all tiles should align perfectly.

Q: "Can I use this with custom shaders?"
A: Yes! The alignment only adjusts the TerrainLayer's tileOffset
   property. As long as your shader respects this property, it
   will work.

Q: "Does this work with normal/height maps?"
A: Yes! All texture channels (diffuse, normal, mask) are aligned
   together since they share the same tileOffset.
